# 同步 Skill 协议

## 功能

以当前项目的 `.claude/skills/` 为源，扫描本机其他 git 仓库，AI 分析目标项目特点后，智能适配协议内容并同步。

**不是文件复制工具**，AI 必须理解目标项目后适配内容。

## 触发模式

### 模式 A — 指定同步内容

用户指定要同步的具体内容，例如：
- "把文件删除规范同步到其他项目"
- "把 commit 规范同步到其他项目的 protocol-dev"

### 模式 B — 全量差异分析

用户未指定具体内容，例如：
- "同步一下协议"
- "检查其他项目的协议差异"

## 执行流程

### 第一步：扫描目标仓库

1. 当前项目为源（source of truth）
2. 扫描 `~/Desktop/Program/` 下所有 git 仓库：

```bash
find ~/Desktop/Program -maxdepth 4 -name ".git" | while read gitpath; do
  repo_dir=$(dirname "$gitpath")
  if [ "$repo_dir" != "<当前项目绝对路径>" ] && [ -d "$repo_dir/.claude/skills" ]; then
    echo "$repo_dir"
  fi
done
```

3. 列出找到的目标仓库，供用户确认范围

### 第二步：分析目标项目

对每个目标仓库，**必须**读取以下内容来理解项目：
- `CLAUDE.md`（项目说明、技术栈、约束）
- `.claude/skills/` 下所有 skill 的 SKILL.md

### 第三步：根据模式执行

**模式 A（指定内容）**：
1. 定位源项目中用户指定的内容（哪个 skill 的哪个章节/规范）
2. 读取目标项目对应的 skill 协议
3. 分析目标项目特点（技术栈、语言、框架等）
4. 适配内容：保留核心规范精神，调整项目特定的细节
5. 展示适配后的修改方案（改哪个文件、改什么位置、改成什么）
6. **等待用户确认后才写入**

**模式 B（全量差异）**：
1. 逐个对比源项目和目标项目的 skill 协议差异
2. 生成差异报告（哪些章节/规范不同、哪些是源端新增的）
3. 展示差异报告，等用户指定要同步哪些内容
4. 用户指定后，转入模式 A 的适配流程

### 第四步：写入与验证

用户确认后：
1. 写入修改到目标项目
2. 展示变更摘要（修改了哪些文件、具体改了什么）

## 适配原则（强制）

- **禁止生搬硬套**：不能把 Rust 项目的协议原样复制到 iOS 项目
- **保留核心精神**：规范的核心意图必须保留（如"删除必须可恢复"）
- **适配项目特点**：根据目标项目的技术栈调整具体实现
- **尊重已有内容**：如果目标项目已有类似规范但表述不同，优先保留目标项目的表述风格
- **每个项目单独确认**：不能批量写入，必须逐个展示方案、逐个确认

## 安全规范

- 遵循 protocol-dev 的"先谋后动"原则
- 所有修改必须先展示方案，等待用户确认后才写入
- 如果目标项目有源端不存在的自定义规范，不得删除或覆盖
